Event-driven architecture (EDA) is a software design model built around the publication, capture, processing and storage of events.
-From IBM

This type of system is meant to be used in the could. Where you have events and based on those events code that needs to do something. This architecture is driven by the pricing of the cloud resources and to manage the cots at the end.

But not only. This type of system has been used all across applications for years. Now its used to fit projects that are not meant for EDA into the EDA structure due to cloud resource management.

Data streaming revolves around the [real-time](https://www.ibm.com/think/topics/real-time-data) flow of events, which are data structures that record any occurrence or change in the system or environment. Examples include a user adding an item to their shopping cart or requesting a password reset. A [stream](https://www.ibm.com/think/topics/streaming-data) (also called a data stream or [event stream](https://www.ibm.com/think/topics/event-streaming)) is the continuous delivery of those events.
-From IBM
